Parte 1 — Perché Creare Questo Sistema

Un professionista accumula nel tempo una grande quantità di materiali dispersi in cartelle diverse, email e appunti. Il problema non è solo conservarli, ma ritrovarli, collegarli e riutilizzarli quando servono.

1.1 L'esigenza iniziale

Immagina uno studio professionale o un'azienda dopo anni di attività:

  • 📁 Cartelle Windows con file sparsi senza logica
  • 📧 Email importanti difficili da ritrovare
  • 📄 PDF di riferimento salvati senza criterio
  • 📝 Appunti personali su riunioni e decisioni
  • 🔗 Nessun collegamento tra clienti, progetti e concetti
  • ❓ Dipendenza dalla memoria di chi conosce la storia

1.2 Problemi del metodo tradizionale

❌ Informazioni disperse

Lo stesso cliente ha dati in Word, PDF, email, CRM. Nessun collegamento tra loro.

❌ Ricerca lenta

Per trovare un'analisi fatta 6 mesi fa, si perde tempo a cercare tra cartelle.

❌ Conoscenza volatile

Se una persona se ne va, la conoscenza va persa. Non è documentata.

❌ Assenza di pattern

È impossibile vedere relazioni tra clienti, settori, problemi ricorrenti.

1.3 Obiettivo del sistema

L'obiettivo non è creare un semplice archivio, ma costruire un ambiente dove la conoscenza sia:

  • ✅ Raccolta ordinatamente in un unico luogo
  • ✅ Collegata semanticamente (cliente → progetti → concetti)
  • ✅ Aggiornata continuamente senza sforzo manuale
  • ✅ Interrogabile intelligentemente via Claude
  • ✅ Consultabile visivamente via Obsidian
  • ✅ Controllata dall'utente (mai fuori controllo)

1.4 Risultato atteso

ℹ️ Visione finale

Quando il sistema è maturo, il professionista può fare domande come:

  • "Quali clienti hanno problemi di cash flow?" → Claude legge il vault e risponde con dati
  • "Quali procedure usiamo per questo processo?" → Obsidian mostra il link a processi correlati
  • "Che cosa abbiamo imparato da questo tipo di cliente?" → Claude analizza i pattern

Parte 2 — Architettura del Sistema

Il sistema è composto da 4 strati che lavorano insieme:

2.1 I quattro strati

2.2 Cos'è Obsidian

Obsidian è un editor di file markdown locale, connesso a una cartella del PC.

💡 Ruolo di Obsidian

Non è un software cloud. È un'interfaccia locale che legge i tuoi file markdown da disco. La vera risorsa è la cartella di file — Obsidian è solo la finestra.

Cosa puoi fare con Obsidian:

  • ✏️ Scrivere e editare note in markdown
  • 🔗 Creare link tra note ([[nota-1]])
  • 🔍 Cercare full-text tra tutti i file
  • 📊 Visualizzare grafi di collegamento (quali note si citano)
  • 📁 Organizzare file in cartelle e tag

2.3 Cos'è Claude

Claude è un AI che puoi usare come agente per leggere il vault, organizzarlo e rispondere a domande.

💡 Ruolo di Claude

Claude non modifica il vault da solo. Tu gli dai istruzioni (skill), lui legge i file e suggerisce azioni o risponde domande. Tu decidi cosa applicare.

2.4 Cloud Storage come infrastruttura

Il vault vive in una cartella sincronizzata (OneDrive, Google Drive, Dropbox) perché:

Vantaggio Motivo
✅ Sincronizzato Stesso vault su PC, tablet, altro device
✅ Backup automatico Il provider tiene copie; non perdi dati se il PC si rompe
✅ Condivisibile Puoi dare accesso ad altri (colleghi, collaboratori)
✅ Locale e cloud Obsidian lavora offline, sincronizza quando sei connesso

2.5 MECE come principio organizzativo

MECE = Mutually Exclusive, Collectively Exhaustive (mutualmente esclusivo, collettivamente esaustivo).

Significa: ogni nota ha una sola categoria, e tutte le note sono categorizzate.

Esempio di categorizzazione MECE:

❌ Male:

  • Note per "Clienti"
  • Note per "Progetti"
  • Note sparse senza categoria (BUCO!)
  • Una nota su un cliente che è anche un progetto (DUPLICE!)

✅ Bene:

  • 01-Inbox (nuovi input)
  • 02-Processi (procedure studio)
  • 03-Risorse (knowledge base)
  • 04-Clienti (info cliente + storia)
  • 05-Progetti (attivi)
  • 06-Archivi (completati)

# Istruzioni per Claude — Second Brain

## Ruolo

Sei l'agente che aiuta a mantenere ordinato e aggiornato il Second Brain.

1. **LINT**: Diagnostica

   - Verifica link rotti

2. **INGEST**: Indicizzazione

3. **CLOSE**: Reporting

- 02-Processi
- 03-Risorse
- 04-Clienti
- 05-Progetti
- 06-Archivi

Nessun file deve rimanere fuori da queste categorie.

## Comandi

/second-brain lint
/second-brain ingest
/second-brain close

6.3 Quando lavorare manualmente vs delegare a Claude

Compito Chi lo fa Perché
Aggiungere nuovi file TU (manuale) Solo tu sai cosa è importante e dove va
Creare link tra note TU (manuale) Richiede comprensione semantica umana

Decidere se applicare suggerimenti TU (manuale) Solo tu hai il contesto per dire sì/no

6.4 Errori tipici da evitare

❌ Non categorizzare i file

Se lascio tutto in 01-Inbox/, il vault diventa caos. MECE fallisce.

✅ Soluzione: Dopo 1 settimana, sposta file in categoria corretta.

❌ Usare lo stesso file per due concetti

Es: "Cliente Bianchi + Progetto Alpha" nello stesso file. Viola MECE.

✅ Soluzione: Due file separati, collegati con link.

❌ Non sincronizzare il cloud

Se il cloud non sincronizza, Obsidian vede file vecchi, Claude vede dati stantii.

✅ Soluzione: Controlla l'icona del cloud storage ogni mattina.

❌ Ignorare i report di Claude

Claude segnala "3 file orfani", tu ignori. Il vault degrada.

✅ Soluzione: Leggi report settimanale e applica fix.

6.5 Quando il vault è "maturo"

Dopo ~2-3 mesi di uso sistematico, il vault raggiunge uno stato di "equilibrio dinamico":

  • ✅ 80%+ MECE coverage (pochi file non categorizzati)
  • ✅ <2 duplicati per settimana (Claude li segnala e li fissi subito)
  • ✅ Link density 2-3 link per file (note ben collegate)
  • ✅ Health score >85/100
  • ✅ Tempi di ricerca <1 minuto (Obsidian o Claude)
